# Contributing to Win323

Thank you for your interest in contributing! Here are guidelines to help you get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/win323`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Install dependencies: `npm install`

## Development

### Build
```bash
npm run build
```

### Run in Development Mode
```bash
npm run dev
```

### Run Tests
```bash
npm test
```

### Lint Code
```bash
npm run lint
```

### Format Code
```bash
npm run format
```

## Commit Guidelines

- Use clear, descriptive commit messages
- Reference issues when applicable: `fixes #123`
- Use conventional commits: `feat:`, `fix:`, `docs:`, `chore:`

## Pull Request Process

1. Ensure all tests pass: `npm test`
2. Update documentation if needed
3. Add tests for new features
4. Create a PR with a clear description
5. Reference related issues in the PR description

## Code Style

- Follow TypeScript best practices
- Use ESLint configuration: `npm run lint`
- Format with Prettier: `npm run format`
- Write meaningful variable and function names

## Reporting Issues

When reporting issues, include:
- Clear description of the problem
- Steps to reproduce
- Expected behavior vs. actual behavior
- Your environment (OS, Node version, etc.)

## Feature Requests

Feature requests are welcome! Please describe:
- The use case
- Why it's valuable
- How it might work

Thank you for contributing! 🎉
